### 1.1 计算机的定义

**本质定义**：
> 计算机是一种**可编程的电子设备**，能够接收、存储、处理和输出数据。

**核心特征**：
- ✅ **可编程性**：通过改变程序实现不同功能（与计算器本质区别）
- ✅ **存储程序**：程序和数据都存储在内存中（冯·诺依曼架构）
- ✅ **自动执行**：无需人工干预，按指令顺序执行
- ✅ **通用性**：理论上可以计算任何可计算的问题（图灵完备）

**重要认知**：
- 计算机**不是万能的**，存在理论上不可计算的问题

- 理解计算机的能力边界，才能合理使用计算机

**为什么用晶体管**？

| 特性 | 电子管 | 晶体管 | 根本原因 |
|------|-------|-------|---------|
| 尺寸 | 厘米级 | 纳米级 | 半导体材料特性 |

- 晶体管本质是一个**电子开关**
- 通过电压控制**导通**（1）或**截止**（0）

#### CPU 执行指令的过程

```mermaid
sequenceDiagram
    participant PC as 程序计数器
    participant IM as 指令存储器
    participant CU as 控制单元
    participant ALU as 算术逻辑单元

    REG->>ALU: 传送操作数
    ALU->>ALU: 执行运算
    ALU->>REG: 写回结果

```

**指令周期的五个阶段**：
1. **取指**（IF）：从内存取指令
2. **译码**（ID）：解析指令含义
3. **执行**（EX）：执行运算操作
4. **访存**（MEM）：访问数据内存
5. **写回**（WB）：将结果写回寄存器

### 3.4 为什么 CPU 有这么多核心

```mermaid
graph TD
    A[多核 CPU] --> B[单核瓶颈]
    A --> C[并行计算]
    A --> D[功耗限制]
    
    B --> B1[频率提升遇到<br/>物理极限]
    C --> C1[多任务并行<br/>提升吞吐量]
    D --> D1[频率越高<br/>功耗越大<br/>散热困难]
    
    style A fill:#8e44ad,color:#fff
    style B fill:#e74c3c,color:#fff
    style C fill:#27ae60,color:#fff
    style D fill:#f39c12,color:#fff
```

**根本原因分析**：

| 因素 | 说明 | 影响 |
|------|------|------|
| **频率墙** | 单核频率接近 5GHz 极限 | 无法继续提升频率 |
| **功耗墙** | 功耗与频率立方成正比 | 散热成为瓶颈 |
| **并行需求** | 多任务、多线程应用增多 | 需要并行处理能力 |
| **阿姆达尔定律** | 程序串行部分限制加速比 | 多核提升有限 |

**阿姆达尔定律**：
```
加速比 = 1 / (S + (1-S)/N)
```
其中：S = 串行部分比例，N = 核心数

**关键认知**：即使有无限多核心，加速比也受限于**串行部分比例**。

#### CISC vs RISC 对比

| 特性 | CISC(x86) | RISC(ARM/RISC-V) | 根本原因 |
|------|----------|-----------------|---------|

**趋势**：
- x86 内部也将复杂指令**微操作化**为简单指令
- RISC 和 CISC 的**界限逐渐模糊**
- **RISC-V** 作为开源指令集崛起

## 四、数据表示：为什么是二进制

### 4.1 为什么计算机使用二进制

```mermaid
graph TD
    A[为什么用二进制] --> B[物理实现简单]
    A --> C[抗干扰能力强]
    A --> D[运算规则简单]
    A --> E[与逻辑代数对应]
    
    B --> B1[两种稳定状态<br/>高电平/低电平]
    C --> C1[噪声容限大<br/>0 和 1 易区分]
    D --> D1[布尔代数<br/>逻辑门实现]

    style A fill:#8e44ad,color:#fff

    style C fill:#27ae60,color:#fff

    style E fill:#f39c12,color:#fff
```

**根本原因**：
- **物理层面**：晶体管只有导通/截止两种稳定状态
- **工程层面**：二值系统抗干扰能力最强
- **数学层面**：布尔代数完美对应二进制运算

**对比其他进制**：

| 进制 | 状态数 | 噪声容限 | 实现难度 | 信息密度 |
|------|-------|---------|---------|---------|
| 二进制 | 2 | 50% | 最低 | 最低 |
| 三进制 | 3 | 33% | 中等 | 中等 |
| 十进制 | 10 | 10% | 最高 | 最高 |

**关键认知**：二进制不是最优的**信息密度**选择，但是最优的**可靠性**选择。

### 5.4 分支预测：为什么需要猜

```mermaid
graph TD
    A[分支预测] --> B[问题]
    A --> C[影响]
    A --> D[技术]
    
    B --> B1[遇到条件分支<br/>不知道走哪条路]
    C --> C1[等待条件计算<br/>流水线停顿<br/>性能下降]
    D --> D1[静态预测<br/>动态预测<br/>神经网络预测]
    
    style A fill:#8e44ad,color:#fff
    style B fill:#e74c3c,color:#fff
    style C fill:#f39c12,color:#fff
    style D fill:#27ae60,color:#fff
```

**根本原因**：
- 现代 CPU 流水线很长（10-20 级）
- 分支指令很常见（约 20% 指令）
- 等待分支结果会导致流水线**清空**，损失巨大

**预测准确率**：
- 静态预测：约 60-70%
- 动态预测：约 90-95%
- 现代 CPU（如 Intel）：约 98%

**预测失败的代价**：
```
损失周期数 = 流水线级数 × 预测失败率 × 分支频率
= 15 × 2% × 20% ≈ 0.06 周期/指令
```

- 硬盘访问约 **1000 万周期**
- 缓存命中率决定系统性能

- **CPI**：每条指令的平均时钟周期数

#### 阿姆达尔定律

```mermaid

    A[阿姆达尔定律] --> B[串行部分限制<br/>最大加速比]
    A --> C[并行部分可<br/>随核心数扩展]
    
    B --> B1[10% 串行<br/>最大加速 10 倍<br/>无论多少核心]

**关键认知**：优化**串行部分**比增加核心数更重要。

```

**示例：矩阵遍历**

```
// 缓存友好（行优先）
for (i = 0; i < N; i++)
    for (j = 0; j < N; j++)

for (j = 0; j < N; j++)
    for (i = 0; i < N; i++)
        sum += matrix[i][j];  // 跳跃访问
```

**性能差异**：可达 **10 倍以上**
